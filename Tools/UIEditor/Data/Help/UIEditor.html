1. Начало работы с редактором
  1. Создание нового проекта
    1. В главном меню редактора выбираем File->New project.
    2. В открывшемся файловом диалоге, нужно задать папку, в которой будет храниться проект, а также имя проекта. После этого нажимаем "Save".
    3. В главном меню редактора выбирам File->Project->New Platform. В открывшемся диалоге, необходимо задать имя вашей платформы, а также ее размеры - ширину и высоту в пикселях. После выставления параметров нажимаем "OK". Если действите выполнено успешно - имя платформы должно появиться в блоке "Hierarchy Tree". Можно создавать произвольное количество платформ (Новую платформу можно создать с помощью соответствующей кнопки на панели инструментов).
    4. В главном меню редактора выбирам File->Project->New Screen. Это позволит создать "экран", на котором в дальнейшем будут размещаться элементы управления. В открывшемся диалоге, необходимо выбрать платформу, к которой будет привязан "экран", а также указать его имя. После выставления параметров нажимаем "OK". Новый экран получит размеры, которые соотвествуют размерам платформы к которой он привязан (Для создания экрана также можно воспользоваться соответсвующей кнопкой на панели инструментов).
    5. Для коректной работы с созданым проектом, необходимо сначала его сохранить. В главном меню редактора выбираем File->Save Project. В случае успешного сохранения, в папке проекта (которую мы задали в п.п. 1.2.2) будет создана новая папка (или несколько в зависимости от количества "платформ") имя которой будет совпадать с именем платформы.
  2. Открытие существующего проекта
    1. В главном меню редактора выбираем File->Open project.
    2. В открывшемся файловом диалоге, выбираем папку с проектом и нажимаем "Open". В случае успешной загрузки будет построено дерево объектов, будут созданы платформы и экраны, а текущий экран отобразится в окне редактирования.
    3. Загрузить существующий проект можно также через список ранее открытых проектов - File->RecentProjects, либо же воспользоваться соответствующей кнопкой на панели инструментов.
  3. Установка шрифтов
    1. По умолчанию для новых элементов управления (Control-ов) используется шрифт MyriadPro-Regular.otf. Он находится во внутренних ресурсах приложения и поменять его нельзя. Для того чтобы получить возможность менять шрифт элементов управления - необходимо установить новые шрифты.
    2. Обычные шрифты(true type), которые могут быть использованы в редакторе, находятся в папке Fonts (кроме шрифта по умолчанию).
    3. Для того чтобы установить новые шрифты, нужно их скопировать в папку <Название проекта>\Data\Fonts (если по указаному пути нет такой папки - нужно создать новую).
    4. В редакторе используются два типа шрифтов - обычные TrueType шрифты (поддерживаются шрифты с расширением *.ttf, *.otf, *.fon, *.fnt) и графические шрифты. Графические шрифты состоят из трех файлов - определение шрифта (Font Definition - расширение *.def), текстура шрифта (Font Sprite - расширение *.txt) и непосредственно графического файла, который привязан к текстуре.
    5. Что бы посмотреть какие шрифты доступны к использованию, в главном меню редактора выбираем Tools->Font Manager. В открывшемся диалоге, будет список установленных шрифтов и их тип (обычный - Basic или графический - Graphics).
  4. Установка графических шрифтов и текстур
    1. Файлы текстур (в формате psd) необходимо скопировать в папку <Название проекта>\DataSource\Gfx (если такой папки нет - необходимо создать ее).
    2. Файлы текстур для графических шрифтов (в формате psd) необходимо скопировать в папку <Название проекта>\DataSource\Fonts
    3. В редактор встроен упаковщих текстур. Он запускается в момент когда пользователь собирается установить текстуру на элемент управления или поменять шрифт текста элемента.
2. Главное меню редактора
  1. File
    1. New Project - открывает файловый диалог сохранения нового проекта.
    2. Save Project - сохраняет текущий проект. Команда недоступна если нет активного проекта.
    3. Open Project - открывает существующий проект.
    4. Close Project - закрывает текущий проект. Команда недоступна, если нет активного проекта.
    5. Project - подменю для управления активным проектом.
      • New Platform - создает новую платформу для текущего проекта. Команда недоступна если нет активного проекта.
      • New Screen - создает новый экран редактирования для выбраной платформы. Команда недоступна если нет активного проекта и платформы.
    6. Recent projects - список последних 5 проектов которые были открыты.
  2. Edit
    1. Undo - отменяет последнее действие.
    2. Redo - возобновляет последнее действие, если оно было отменено.
    3. Special characters - открывает диалог выбора специальных символов, которые можно поместить в текст элементов управления.
  3. View
    1. Hierarchy Tree - показывает/прячет блок дерева элементов управления редактора.
    2. Library - показывает/прячет блок библиотеки элементов управления редактора.
    3. Properties - показывает/прячет блок свойств элементов управления редактора.
  4. Tools
    1. Font Manager - открывает диалог управления шрифтов.
    2. Localization Manager - открывает диалог управления локализациями.
  5. Help
    1. Help - открывает файл-инструкцию (этот документ).
3. Дерево элементов
  1. В блоке Hierarchy Tree представлено дерево элементов.
  2. Навигация в дереве элементов соответствует стандартной схеме навигации в подобных структурах.
  3. Родительским элементом для всех веток дерева является платформа. Второй веткой дерева (дочерний элемент платформы) является экран. Экран в свою очередь является родительским элементом для всех веток дерева, которые соответствуют элементам управления.
  4. Для элементов дерева через контекстное меню доступны следующие действия:
    1. Delete - удаляет выбраный элемент, все вложенные элементы будут также удалены.
    2. Copy - копирует выбранный элемент в буфер обмена.
    3. Paste - вставляет из буфера обмена элемент. Команда доступна только для элемента дерева типа "экран".
    4. Create screen - открывает диалог создания нового экрана. Команда доступна только для элемента дерева типа "платформа".
  5. Можно менять порядок и вложенность элементов управления в дереве используя механизм drag and drop. Любой элемент управления может быть вложен в другой элемент.
  6. Также доступно копирование элементов с помощью механизма drag and drop. Для этого достаточно начать перетаскивание элемента с зажатой клавишей Control.
4. Область редактирования и создание элементов управления (Controls)
  1. Создание и удаление элементов управления.
    1. Прежде чем создавать элемент управления, в редакторе должен быть открыт проект, создана платформа и экран.
    2. Для создания нового элемента управления необходимо перетащить соответствующий элемент из блока "Library" непосредственно в область экрана.
    3. Новый созданный элемент отобразится в области экрана и станет текущим (Текущий выбранный элемент выделен красной прямоугольной рамкой).
    4. Что бы удалить выбраный элемент необходимо нажать клавишу "del".
  2. Перемещение и изменение размеров элементов управления
    1. Расположение элемента управления на экране можно менять используя механизм drag and drop, либо же стрелки управления.
    2. Для изменения размера выделенного элемента управления, необходимо подвести курсор к его границе. Появятся стрелки изменения размеров.
    3. Для изменения масштаба элементов на выбраном экране, необходимо воспользоваться соответствующей шкалой вверху области редактирования. Также можно воспользоваться комбинацией Control + MouseWheel (Cmd + MouseWheel для Mac).
    4. Выделение нескольких элементов управления на экране осуществляется комбинацией SHIFT + щелчок левой кнопки мышки, либо же обычным выделением рамкой при зажатой левой кнопке мышки.
  3. Копирование элементов управления
    1. Для копирования выделенных элементов на экране, нужно зажать левую кнопку мышки и клавишу Control(Cmd для Mac) и начать перетаскивание.
    2. Скопированные элементы автоматически станут текущими и появятся в дереве эелементов.
    3. Для завершения копирования необходимо отпустить левкую кнопку мышки.
    4. В случае копирования элементов, у которых разные "родители" в дереве, все выделенные элементы будут скопированых в первый доступный "родительский" элемент.
5. Свойства элементов управления
  1. Блок Properties содержит в себе набор виджетов, в каждом из которых сгруппированы по типу свойства элементов управления. На текущий момент доступно 6 групп:
    1. Control - общие свойства элемента управления.
    2. Rect - свойства отвечающие за местоположение экране и размеры элемента управления.
    3. State - выпадающий список для установки "состояния" свойств элемента управления (например для элемента управления UIButton для разных состояний можно выставить различный шрифт и текст, цвет заливки и еще ряд свойств).
    4. Text - свойства для установка текста, шрифта и размера шрифта элемента управления. Этот блок свойств недоступен для элемента управления UIControl. Также блок имеет некоторые различия для элементов UIButton/UIStaticText и UITextField.
    5. Background - свойства для установки цвета заливки, типа заливки, текстуры, внутренней рамки, выравнивания текста и т.п.
    6. Flags - логические/булевские свойства элемента управления.
  2. Блок свойств Control
    1. Class - имя класса элемента управления. Только чтение.
    2. Name - имя элемента управления.
    3. Tag - тег элемента управления.
  3. Блок свойств Rect
    1. Relative X: Y: - Относительные координаты крайней верней левой точки элемента управления на экране.
    2. Absolute X: Y: - Абсолютные координаты крайней верхней левой точки элемента управления на экране.
    3. Size X: Y: - Размеры элемента управления (ширина и высота).
    4. Pivot Point X: Y: - Точка поворота содержимого элемента управления.
    5. Angle - Угол поворта содержимого элемента управления от -360 до 360 (в градусах). При изменении угла у родительского элемента управления, угол наклона содержимого также меняется у всех дочерних элементов.
  4. Блок свойств Aligns
    1. Left: - отступ от левого края родительского элемента управления.
    2. Hcenter: - отступ от центра по горизонтали родительского элемента управления, положительные значения – сдвиг вправо, отрицательные – влево
    3. Right: - отступ от правого края родительского элемента управления.
    4. Top: - отступ от верхнего края родительского элемента управления.<
    5. VCenter: - отступ от центра по вертикали родительского элемента управления, положительные значения – сдвиг вниз, отрицательные – вверх
    6. Bottom: - отступ от нижнего края родительского элемента управления.<
  5. Блок State
    1. Выпадающий список с доступными "состояниями" элемента управления.
    2. Состояние элемента управления могут быть такими: Normal, Pressed Inside, Pressed Outside, Disabled, Selected, Hover
    3. Для каждого состояния можно задать различный текст, цвет заливки, текстуру, шрифт, а также - Draw Type, Color Inherit, Align
  6. Блок свойств Text
    1. Localization Key - Ключ локализации. При заданом ключе, редактор попытается вычитать по этому ключу соответствующий ему текст из папки локализации для выбраного языка.
    2. Control Text - Отображает текущий текст элемента управления. Доступна только для чтения.
    3. "Set text for all states" - в отмеченом виде позволяет устанавливать текст для всех состояний элемента управления. В противном случае текст будет установлен только для текущего состояния.
    4. Font - Кнопка выбор шрифта элемента управления. В текст кнопки отображается относительный путь к файлу шрифта (в случае графического шрифта будет показан еще относительный путь к файлу текстуры).
    5. Font size - выставление размера шрифта элемента управления.
  7. Блок свойств Background
    1. Color - кнопка выбора цвета заливки элемента управления. При нажатии будет открыт стандартный диалог выбора цвета. Выбраный цвет также отобразится на кнопке.
    2. Sprite - выбор текстуры для элемента управления. В текстовом поле отображется путь к файлу текстуры, кнопка "..." открывает файловый диалог выбора файла текстуры, кнопка "х" сбрасывает текстуру элемента управления.
    3. Frame - фрейм элемента управления.
    4. Draw Type - управление содержимым элемента управления.
    5. Color Inherit - параметр наложения слоев сореджимого элемента управления.
    6. Align - выравнивание содержимого элемента управления.
  8. Блок свойств Flags
    1. Visible - прячет/показывает элемент управления на экране.
    2. Enabled - включает/выключает элемент управления.
    3. Input - позволяет/запрещает редактирование содержимого элемента управления.
    4. Clip Contents - позволяет/запрещает перекрывать события дочерних элементов управления.
    5. Selected - выделяет/снимает выделение с элемента управления.
  9. Изменение текста элемента управления
    1. Что бы изменить текст элементов управления UIButton и UIStaticText необходимо задать ключ локализации. Напрямую изменить текст этих элементов нельзя.
    2. В случае если не установлен путь к папке с файлами локализации или введенный ключ не обнаружен, отобразиться введеный текст.
    3. При заданом флаге "Set text for all states" введеный текст будет установлен для всех состояний элемента управления, в противном случае только в том которое выбрано в выпадающем списке блока State.
  10. Изменение шрифта элемента управления
    1. Что бы изменить шрифт элемента управления необходимо нажать на соответствующую кнопку в блоке свойств Text.
    2. В открывшемся диалоге, необходимо выбрать из списка нужный шрифт и нажать кнопку "OK".
    3. В случае выбора графического шрифта, будет открыт дополнительный файловый диалог с просьбой указать текстуру шрифта.
    4. Изменения вступят в силу сразу же, а кнопка выбора шрифта отобразит относительный путь к шрифту (и текстуре в случае графического).
    5. Рекомендуется использовать только те шрифты который установлены в папку Fonts платформы проекта.
  11. Изменение текстуры элемента управления
    1. Что бы изменить текстуру элемента управления необходимо нажать на соответствующую кнопку в блоке свойств Background.
    2. В открывшемся файловом диалоге, необходимо выбрать файл текстуры (формат *.txt).
    3. Для удаления текстуры элемента управления, необходимо нажать соответствующую кнопку в блоке свойств Background.
Панель инструментов
  1. На панели инструментов продублированны основные функции редактора.
6. Диалог Font Manager
  1. Font Manager предназначен для отображения списка установленных шрифтов, а также для смены шрифта элемента управления.
  2. Диалог управления шрифтами можно открыть двумя способами - через главное меню или через соответствующую кнопку в блоке свойств для конкретного элемента управления.
  3. Во втором случае в диалоге будет доступна кнопка "OK", по нажатию которой, у текущего элемента управления будет установлен шрифт выбранный в диалоге.
  4. В случае выбора графического шрифта будет открыт дополнительный файловый диалог для выбора файла с текстурой.
7. Диалог Localization Manager
  1. Localization Manager преназначен для установки папки локализации и выбора текущего языка.
  2. Для выбора папки локализации нужно нажать кнопку "Choose directory". В открывшемся файловом диалоге нужно выбрать директорию локализации. После этого станет доступна смена языков.
  3. Для смены текущего языка необходимо выбрать нужный тип из выпадающего списка. Изменения вступят в силу сразу же.
